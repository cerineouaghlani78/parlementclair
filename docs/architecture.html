Flux d'un tour (streaming)

  1. L'utilisateur tape un message ; React l'ajoute au state messages et affiche une bulle assistant vide.
  2. React envoie tout l'historique : POST /api/chat avec { messages } (proxifié vers Flask par Vite).
  3. Flask valide le body, préfixe le prompt système : [system] + messages.
  4. Flask appelle Groq en stream=True et ouvre une réponse text/event-stream.
  5. Groq génère et renvoie la réponse en chunks au fur et à mesure.
  6. Pour chaque chunk, Flask yield data: <token> vers le navigateur.
  7. React lit le flux et concatène les tokens dans la bulle assistant (effet temps réel).
  8. À la fin du flux, la bulle assistant est finalisée dans le state — prête à être renvoyée au tour suivant, ce qui préserve le contexte.
